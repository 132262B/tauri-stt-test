# 00 — Product Seed & Scope (온디바이스 회의 전사 앱)

> 출처: `/symposium:interview-harness` (socrates → evolve-step → ontology, 2 cycles, 수렴 1.0).
> 원본 인터뷰 기록은 `.symposium/scratch/` 참고(gitignore). 이 문서가 canonical 사양이다.

## Goal

Mac·iOS용 Tauri 테스트 앱(Rust 백엔드)을 만들어, 마이크와 시스템 오디오를 캡처하고
온디바이스 전용 추론(VAD · 온라인 화자분리 · 교체형 ASR 백엔드)으로 회의를 **실시간 화자 라벨
전사**하며, 동작 중 **메모리·자원 사용량을 화면에서** 확인할 수 있게 한다.

## Constraints

1. 모든 추론은 런타임에 **온디바이스로만** 수행한다(클라우드 API 호출 0). 모델 가중치는 인터넷 다운로드만 허용.
2. 백엔드는 **Rust**로 구현하고 **Tauri**로 앱을 만든다. 우선순위는 **Mac → iOS**.
3. ASR 백엔드는 **교체 가능**해야 하며 **MLX Whisper(Apple Silicon)**, **Voxtral(MLX)**, **qwen ASR**을 지원한다.
4. VAD·화자분리는 기존 **whisper-live-kit**를 참고하여 개발한다.
5. 전사는 **실시간 라이브 전용**(온라인 화자분리)이며, 회의 종료 후 일괄 재정리/확정 단계는 두지 않는다.
6. 산출물은 **화자 라벨이 달린 전사 텍스트까지**이며, 요약·회의록 가공은 범위 밖.
7. 오디오 입력은 **마이크 + 시스템 오디오** 모두 지원. 시스템 오디오는 OS 권한 기반이며, **iOS에서도 시스템/통화 오디오 캡처 방법을 모색**한다.
8. 모든 커밋 메시지/메타데이터에 **claude(AI 도구) 표기를 남기지 않는다.**
9. **작업 단위마다 커밋**하여 이력을 관리한다.
10. 전사 주 대상 언어는 **한국어·영어 이중언어(코드스위칭)**, 다국어 모델·자동 언어 감지 고려.
11. 세션 종료 후 전사 결과를 **파일(txt/srt/json 등)로 저장·내보내기** 가능해야 한다.
12. 권한 거부·모델 다운로드 실패·발열/배터리(특히 iOS) 등 **실패·엣지 상황**을 안전하게 처리한다.
13. 화자 분리는 익명 라벨(Speaker 1/2)을 넘어 **화자 등록·식별(enrollment/identification)**까지 확장한다.

## Acceptance Criteria

- [ ] 인터넷 차단 상태(모델 다운로드 완료 후)에서도 전사가 완전히 동작한다.
- [ ] Mac에서 마이크+시스템 오디오 입력으로 '누가 무엇을 말했는지' 화자 라벨 전사가 진행 중 실시간 화면 누적된다.
- [ ] MLX Whisper / Voxtral / qwen ASR 세 백엔드를 앱에서 전환할 수 있고 각각 실제 전사가 동작한다.
- [ ] 현재 메모리(가능하면 CPU·지연 포함) 사용량이 앱 화면에 실시간 표시된다.
- [ ] 서로 다른 화자 2명 이상을 서로 다른 라벨로 구분 표시한다.
- [ ] VAD·화자분리가 기존 whisper-live-kit와 동등한 수준으로 재현된다.
- [ ] 말한 후 전사 텍스트가 체감상 수 초 이내 저지연으로 나타난다.

## Ontology (의미 경계)

- **idea**: (= Goal)
- **boundary**: "온디바이스 검증 데모 전체."
  - 경계 **안**: 모델 다운로드·로딩, 온디바이스 추론, ASR 백엔드 실행 중 전환, 자원 모니터링 — 즉 *클라우드 0 스택이 Apple Silicon/iOS에서 동작함*을 보이는 것까지.
  - 경계 **밖**: 실제 회의 운영(일정·참가자 관리), 전사 이후 가공(요약·분석·검색).
- **properties** (경계 안에 반드시 있어야 할 성질):
  1. **관찰 가능성** — 메모리·CPU·지연 등 자원 상태가 항상 화면으로 드러난다.
  2. **오프라인성** — 런타임에 네트워크 없이 동작한다(클라우드 0).
  3. **교체 가능성** — ASR 백엔드를 실행 중에 바꿔 비교할 수 있다.

## 명시적으로 범위 밖 (인터뷰에서 기각됨)

- LLM 후처리(요약·회의록·액션아이템 추출).
- 사후 2단계 확정(회의 종료 후 전체 오디오 재처리).
- 녹음 파일 import 전사, 비-Apple 플랫폼(Android/Windows/Linux) 1차 타깃, 클라우드 폴백.
  - (단, 저장소에는 이미 Tauri Android gen이 존재 — 1차 범위에서는 다루지 않음.)
