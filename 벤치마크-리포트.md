# 온디바이스 STT 모델 전수 벤치마크 — 제품팀 주간 스프린트 회의

> 측정 음성: `회의-01-제품팀-주간스프린트.md`를 TTS로 합성한 **약 10분(599초)** 회의(화자 4명).
> 정답: 위 대본의 화자 대사(정규화 3,085자). 측정일: 2026-06-25, Apple Silicon(Metal/Accelerate), 전부 온디바이스·in-process(Python/Node 0).

## TL;DR — 결론부터

- **정확도 1위는 Qwen3-ASR**: 0.6B가 CER **2.9%**(정확도 97.1%), 1.7B가 **2.6%**(97.4%). Whisper 계열을 압도(한국어 회의 기준).
- **실시간(스트리밍) 가능 모델은 사실상 tiny·base뿐**: 1초 틱 안에 추론이 끝나는 건 tiny(174ms)·base(248ms). small은 경계선, **turbo/large/Qwen은 현재 스트리밍 설계에선 실시간 미달**.
- **정확도 ↔ 실시간은 트레이드오프**: 가장 정확한 Qwen은 가장 느리고(틱 2.5~5.4s), 가장 빠른 tiny/base는 정확도가 낮다(72%대).
- **Q5_0 turbo의 가치는 "속도"가 아니라 "크기"**: 정확도는 f16 turbo와 **동일(23.7%)**, 크기는 1.5G→**547M(1/3)**. 단 Metal에선 양자화 해제 오버헤드로 **속도는 f16과 거의 같음**(틱 1.3s).
- **large-v3(non-turbo)는 장시간 단일패스에서 환각 루프**("네 이번트는 뭐지?" 수십 회 반복) → CER 79%. **turbo를 쓸 것**.
- **화자분리는 현재 과분할(22명 검출, 정답 4명)** — 튜닝 필요(아래 진단).

**추천**: 한국어 회의 정확도가 목적이면 **Qwen3-ASR-0.6B**(거의 실시간은 아니지만 정확도 압도적). 실시간 자막이 목적이면 **base**. 균형으로는 **small**. turbo가 필요하면 용량 때문에 **Q5_0**를 권장.

- **Qwen의 압도적 우위**: "회의를 시작하겠습니다 / 첫 번째 안건은 / QA랑 데모 일정" 등 거의 오류 없음. Whisper는 "정각→전각, 데모→대모, 로그인→소구인, 이번 스프린트→이번트" 식 한국어 오인식이 많음.
- **large-v3 79%는 진짜 성능이 아님**: 10분 음성을 한 번에 디코딩하니 후반부에서 `"네 이번트는 뭐지?"`를 수십 번 반복하는 **whisper large-v3 특유의 반복 환각**이 발생. 동일 계열인 turbo는 이 현상이 없음 → **turbo 권장, large-v3 비권장**.

## 4. 화자분리(diarization) — ⚠️ 현재 과분할, 튜닝 필요

CAM++(sherpa-onnx) 임베딩 + 코사인 임계 0.5로 측정(ASR 모델 무관, base로 대표 측정).

- **정답 4명(박지민/정우성/한소희/김도현) → 검출 22명, 화자 전환 169회(169라인 = 매 라인 전환)**.
- 즉 **거의 모든 발화마다 새 화자로 인식**되는 과분할 상태.

라벨 전사 샘플:
```
[화자 1] 0:00~0:20  자 다들 ... 다 챙기셨죠? 네, 챙기셨죠.
[화자 2] 0:20~0:21  고생겼습니다.
[화자 3] 0:20~0:34  어려우일은 카페인이 ... 우성님 지난주에 맡으
[화자 1] 0:34~0:35  로그인
[화자 4] 0:36~0:47  됐어요? ... 잘 돌아갑니다.
[화자 5] 0:47~0:53  오 좋네요. ...
[화자 6] 0:56~0:56  동안
```

**원인 진단**
1. **타임스탬프 부정확**: SelfStreaming은 단어 타임스탬프가 윈도우 비례 "근사값"이라, 화자 임베딩용 오디오 구간(끝 2초)이 실제 발화와 어긋남 → 매번 다른 임베딩.
2. **짧은 확정 배치**: 확정 단위가 짧아(1~2단어) 0.5초 미만이면 임베딩 자체가 불안정.
3. **임계값**: 0.5에서 같은 화자도 매칭 실패 → 계속 새 화자 생성.

**개선 방향**(별도 작업): VAD 기반 발화 세그먼트로 끊어 ≥1.5초 구간에서만 임베딩 / 코사인 임계 재튜닝(0.35~0.45) / 화자 수 상한·온라인 클러스터링 안정화. → 정확도보다 화자분리가 이 앱의 다음 핵심 개선 과제.

## 5. 경쟁 앱 `alt` 분석 (참고)

`~/Library/Logs/alt/main.log` + 모델 디렉터리 분석 결과:

- **엔진**: 우리와 동일한 **whisper.cpp**.
- **모델**: **Whisper large-v3-turbo Q5_0**(`ggml-large-v3-turbo-q5_0.bin`, 574M) — 이번에 우리도 추가해 동일 측정함.
- **인코더 가속**: **CoreML**(`...-encoder.mlmodelc`, Float16) → Apple Neural Engine/GPU. 우리는 Metal만.
- **다운로드 출처**: 인코더 `altalt-org/whisper-large-v3-16bit-encoder-coreml`, 가중치 `ggerganov/whisper.cpp`.
- **화자분리 없음**(`transcriptionOnly: true`), 언어 자동감지, 번역 기능 보유(off).

**시사점**: alt는 "turbo Q5_0 + CoreML 인코더, 전사 전용". 우리 앱이 이미 더 넓다(화자분리·Qwen·모델 4종+). alt에서 배울 점은 ① **Q5_0로 turbo 경량화**(이번 반영), ② **CoreML 인코더로 ANE 가속**(whisper-rs `coreml` 피처 — 향후 적용 시 turbo 틱 단축 기대).
